## 🎯 规范亮点

### 1. **按需加载设计** ⭐⭐⭐⭐⭐

```markdown
| 场景 | 阅读文档 |
|------|---------|
| 开始新任务 | quick-start.md |
| 编写代码 | coding-guidelines.md |
| UI 开发 | ui-ux-guidelines.md + 组件库文档 ⭐ |
```

**优点**:

- AI 只需阅读主文件
- 按需加载详细文档
- 减少认知负担
- 提高查找效率

---

### 2. **项目治理规范** ⭐⭐⭐⭐⭐

```markdown
### 1. 脚本存放规范
- 严禁在 backend/ 根目录创建 .py 脚本
- 所有脚本必须存放在 backend/scripts/

### 2. 操作入口规范
- 严禁直接运行 python xxx.py
- 所有运维操作必须封装在 Makefile

### 3. 配置唯一性
- 配置必须在 config.py 或 .env 中定义
- 严禁在脚本中硬编码
```

**优点**:

- 防止配置漂移
- 避免脚本混乱
- 确保可维护性

---

### 3. **Prompt 配置化规范** ⭐⭐⭐⭐⭐

```markdown
核心原则: 所有 AI Prompt 必须配置化

✅ 必须:
- 所有 Prompt 独立成文件
- 在业务代码中通过 import 引用

❌ 禁止:
- 在业务逻辑中硬编码 Prompt
- 使用字符串拼接构建 Prompt
```

**优点**:

- 易于调整和优化
- 支持版本控制
- 便于 A/B 测试

---

### 4. **组件库使用规范** ⭐⭐⭐⭐⭐ (新增)

```markdown
- ✅ 必须优先使用项目组件库
- ✅ 组件库不满足需求时,才使用 Element Plus
- ❌ 禁止在业务页面中重复封装已有组件
```

**优点**:

- 统一 UI 风格
- 提高开发效率
- 减少重复代码
- AI 自动遵循
